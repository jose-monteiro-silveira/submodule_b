Digital Certificates

Por meio da API de Digital Certificates você pode consultar, criar, remover ou atualizar seus certificados SSL.

  • 1. Consultar lista de Digital Certificates
  • 2. Consultar dados de um Digital Certificate
  • 3. Deletar um Digital Certificate
  • 4. Criar um novo Digital Certificate
  • 5. Sobrescrever um Digital Certificate
  • 6. Atualizar campos de um Digital Certificate

1. Consultar lista de Digital Certificates

Retorna a lista de Digital Certificates.

GET /digital_certificates

Permissão necessária: View Security Settings

Parâmetro Descrição Tipo de Parâmetro Tipo de Dado
Authorization

(obrigatório) 		A autenticação por Token previamente criado através do endpoint de Criação de Token.

ex.:

Authorization:
583f8a9ca8d6d5ff2cb50f1d3c4d35cb8939f1bf 		header string

Exemplo de Requisição

{% highlight console %} GET /digital_certificates Accept: application/json; version=2 Authorization: Token 583f8a9ca8d6d5ff2cb50f1d3c4d35cb8939f1bf {% endhighlight %}

Exemplo de Resposta

{% highlight console %} HTTP/2 200 {% endhighlight %}
{% highlight console %} [ { "id": 9, "name": "Certificado SSL para www.sitedemonstration.com.br" }, { "id": 344, "name": "Certificado SSL para www.mydomain.com" } ] {% endhighlight %}

Retorna dados de um Digital Certiticate.

GET /digital_certificates/:id

Permissão necessária: View Security Settings

Parâmetro Descrição Tipo de Parâmetro Tipo de Dado
Authorization

(obrigatório) 		A autenticação por Token previamente criado através do endpoint de Criação de Token.

ex.:

Authorization:
583f8a9ca8d6d5ff2cb50f1d3c4d35cb8939f1bf 		header string
:id

(obrigatório) 		Id do Digital Certificate a ser consultado. Para obter o Id de um digital certificate, consulte a Lista de Digital Certificates. path number

Exemplo de Requisição

3. Deletar um Digital Certificate

Remove um Digital Certificate. O certificado não pode estar em uso para ser removido.

DELETE /digital_certificates/:id

Permissão necessária: Edit Security Settings

Parâmetro Descrição Tipo de Parâmetro Tipo de Dado
Authorization

(obrigatório) 		A autenticação por Token previamente criado através do endpoint de Criação de Token.

ex.:

Authorization:
583f8a9ca8d6d5ff2cb50f1d3c4d35cb8939f1bf 		header string
:id

(obrigatório) 		Id do Digital Certificate a ser consultado. Para obter o Id de um digital certificate, consulte a Lista de Digital Certificates. path number

Exemplo de Requisição

{% highlight console %} DELETE /digital_certificates/344 Accept: application/json; version=2 Authorization: Token 583f8a9ca8d6d5ff2cb50f1d3c4d35cb8939f1bf {% endhighlight %}

Exemplo de Resposta

{% highlight console %} HTTP/2 204 {% endhighlight %}

4. Criar um novo Digital Certificate

Faz o upload de um digital certificate para a Azion.

POST /digital_certificates

Permissão necessária: Edit Security Settings

Parâmetro Descrição Tipo de Parâmetro Tipo de Dado
Authorization

(obrigatório) 		A autenticação por Token previamente criado através do endpoint de Criação de Token.

ex.:

Authorization:
583f8a9ca8d6d5ff2cb50f1d3c4d35cb8939f1bf 		header string
Content-Type

(obrigatório) 		O tipo de codificação utilizada no Body (application/json).

ex.:

Content-Type: application/json 		header string
Certificado

(obrigatório) 		O certificado que se deseja criar deve ser enviado codificado no formato json com os seguintes campos:

  • name (string): campo obrigatório que define um nome para o certificado digital.
  • certificate (string): o certificado é o arquivo que você recebeu de sua Certificate Authority (CA), no formato ASCII PEM, substituindo-se todas as quebras de linha por “\n”. O certificado deve iniciar pelo marcador “----BEGIN CERTIFICATE-----” e terminar pelo marcador “-----END CERTIFICATE-
  • private_key (string): a chave privada é o arquivo que você utilizou para gerar a requisição CSR que enviou para sua CA, substituindo-se todas as quebras de linha por “\n”. A chave privada deve iniciar pelo marcador “----BEGIN RSA PRIVATE KEY-----” e terminar pelo marcador “-----END RSA PRIVATE KEY-----“.
body json

5. Sobrescrever um Digital Certificate

Sobrescreve todos os campos de um Digital Certificate, preservando o id. Considere utilizar o método PATCH no lugar de PUT, para atualizar somente alguns campos de um Digital Certificate, sem alterar o valor dos demais.

PUT /digital_certificates/:id

Permissão necessária: Edit Security Settings

Parâmetro Descrição Tipo de Parâmetro Tipo de Dado
Authorization

(obrigatório) 		A autenticação por Token previamente criado através do endpoint de Criação de Token.

ex.:

Authorization:
583f8a9ca8d6d5ff2cb50f1d3c4d35cb8939f1bf 		header string
Content-Type

(obrigatório) 		O tipo de codificação utilizada no Body (application/json).

ex.:

Content-Type: application/json 		header string
:id

(obrigatório) 		Id do Digital Certificate a ser consultado. Para obter o Id de um digital certificate, consulte a Lista de Digital Certificates. path number
Certificado

(obrigatório) 		Os novos valores de cada campo da Digital Certificate. Se você desejar alterar apenas um dos campos, utilize o método PATCH ao invés do PUT.

  • name (string): campo obrigatório que define um nome para o certificado digital.
  • certificate (string): o certificado é o arquivo que você recebeu de sua Certificate Authority (CA), no formato ASCII PEM, substituindo-se todas as quebras de linha por “\n”. O certificado deve iniciar pelo marcador “----BEGIN CERTIFICATE-----” e terminar pelo marcador “-----END CERTIFICATE----“.
  • private_key (string): a chave privada é o arquivo que você utilizou para gerar a requisição CSR que enviou para sua CA, substituindo-se todas as quebras de linha por “\n”. A chave privada deve iniciar pelo marcador “----BEGIN RSA PRIVATE KEY-----” e terminar pelo marcador “-----END RSA PRIVATE KEY-----“.
body json

6. Atualizar campos de um Digital Certificate

Atualiza um ou mais campos de um Digital Certificate, preservando o valor dos campos não informados. Os campos “certificate” e “private_key” não podem ser atualizados separadamente, sendo sempre enviados em par.

PATCH /digital_certificates/:id

Permissão necessária: Edit Security Settings

Parâmetro Descrição Tipo de Parâmetro Tipo de Dado
Authorization

(obrigatório) 		A autenticação por Token previamente criado através do endpoint de Criação de Token.

ex.:

Authorization:
583f8a9ca8d6d5ff2cb50f1d3c4d35cb8939f1bf 		header string
Content-Type

(obrigatório) 		O tipo de codificação utilizada no Body (application/json).

ex.:

Content-Type: application/json 		header string
:id

(obrigatório) 		Id do Digital Certificate a ser consultado. Para obter o Id de um digital certificate, consulte a Lista de Digital Certificates. path number
Certificado

(obrigatório) 		Devem ser informados um ou mais campos a serem atualizados. Os campos “certificate” e “private_key” não podem ser atualizados individualmente, tendo sempre que ser atualizados em par.

  • name (string): campo obrigatório que define um nome para o certificado digital.
  • certificate (string): o certificado é o arquivo que você recebeu de sua Certificate Authority (CA), no formato ASCII PEM, substituindo-se todas as quebras de linha por “\n”. O certificado deve iniciar pelo marcador “----BEGIN CERTIFICATE-----” e terminar pelo marcador “-----END CERTIFICATE----“.
  • private_key (string): a chave privada é o arquivo que você utilizou para gerar a requisição CSR que enviou para sua CA, substituindo-se todas as quebras de linha por “\n”. A chave privada deve iniciar pelo marcador “----BEGIN RSA PRIVATE KEY-----” e terminar pelo marcador “-----END RSA PRIVATE KEY-----“.
body json
